# 自定义字段映射功能设计规范

## 目标
允许用户完全自定义系统字段，增删字段，并将每个自定义字段映射到 WPS 同步返回的原始业务数据表头，适配不同工厂实际业务表格结构。

## 功能描述

### 核心功能
| 功能 | 说明 |
|------|------|
| ✅ 自动识别表头 | 从最近一次 WPS 同步返回的 raw JSON 中提取所有原始列名 |
| ✅ 增删系统字段 | 用户可以添加自定义新字段，也可以删除不需要的字段 |
| ✅ 自由映射 | 每个系统字段（预设/自定义）下拉选择映射到哪一个 WPS 原始列 |
| ✅ 可见性控制 | 用户勾选哪些字段需要在视图中显示 |
| ✅ 持久化 | 所有配置保存在浏览器 localStorage |
| ✅ 重置功能 | 一键重置为默认映射 |

## 数据结构

### TypeScript 类型定义

```typescript
interface CustomFieldConfig {
  fieldId: string;        // 唯一标识符（用户输入，slug 化）
  displayName: string;    // UI 显示名称
  mappedColumn: string;  // 映射到 WPS 原始表格的哪一列（列名）
  visible: boolean;       // 是否在视图中显示
  isDefault: boolean;    // 是否为系统预设字段（预设字段不能修改 ID/名称，只能删除）
}
```

### localStorage 存储
- Key: `mes_field_mapping_config`
- Value: JSON 序列化的 `CustomFieldConfig[]`

## 工作流程

### 1. 打开设置
```
用户打开设置弹窗 → 切换到「字段映射」标签
  ↓
检查最近一次 sync 是否保存了 raw response（`syncResponse` 状态）
  ↓
  ├── 有 → 从 raw response 提取 range_data 中的所有列名
  │   （从 row=2（表头行）提取所有非空单元格作为列名）
  └── 没有 → 提示文字："请先在「基本设置」中完成一次同步，才能读取表头"
  ↓
填入当前保存的配置（从 localStorage 读取，没有则使用默认配置）
```

### 2. 保存配置
```
用户点击「保存设置」
  ↓
验证：检查是否有重复的 fieldId → 如果重复提示错误
  ↓
保存到 localStorage
  ↓
关闭弹窗，下次同步生效
```

### 3. 同步时应用映射
```
WPS 返回原始数据 → 每一行 row[string[]]
  ↓
先从 raw response 获取表头列表 → 建立 列名 => 列索引 的映射
  ↓
对每个用户配置的字段：
  ├── 如果 mappedColumn 在表头中存在
  │   → value = row[columnIndex]
  │   → task[fieldId] = convertType(value)
  └── 不存在 → value = ''
  ↓
生成 Task[] 返回给应用
```

### 类型转换
- `*Quantity` 结尾的字段 → 自动转换为 `number`
- `*Date` 或 `*Time` 结尾 → parse 为 ISO 日期字符串
- 其他 → 保留 `string`

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 还没同步过没有 raw response | 提示用户先同步 |
| 表头行全为空 | 提示"未能检测到任何表头，请确认同步范围包含表头行" |
| 用户添加重复 fieldId | 保存时提示"字段 ID 不能重复，请修改" |
| mappedColumn 不存在于表头 | 该字段值为空，不影响其他字段 |
| localStorage 读取失败 | 回退到默认配置 |

## 向后兼容

- 原有数据结构不变，只是映射方式改变
